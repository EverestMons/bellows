The intermediate decision detector is a phrase-matching system in `bellows/decisions.py` that scans the NDJSON output stream produced by `claude -p` for assistant narration blocks indicating substantive decisions made mid-step. It works by loading a list of 33 decision-indicating phrases from `INTERMEDIATE_DECISION_PHRASES.md` at the governance root and performing case-insensitive substring matching against each assistant text block in the stream. When one or more blocks match, the detector surfaces them in the verdict request file under an `## Intermediate Decisions Detected` section so the Planner can review agent decisions that would otherwise be invisible between the step prompt and the Output Receipt. The feature was built to address a 2.7% Output Receipt capture rate for mid-step decisions identified in a ground-truth audit — agents routinely narrate scope-changing choices or error recoveries inline, and without the detector these narrations were lost to the Planner. The detector is deterministic and side-effect-free: it either appends matched blocks to the verdict request or omits the section entirely if nothing matches, keeping Layer 1 thin and judgment-free.
