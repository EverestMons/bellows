## Context

Per diagnostic findings (cite per Rule 27): three Planner overrides on 2026-05-23 (`executable-bellows-config-split-2026-05-23` Steps 1, 2, 3) were caused by abs-vs-rel path-form mismatch in `_gate_deposit_exists`. The bug has two intertwined components, both addressed in this single executable:

- **Component A** — Missing normalization in `agent_declared` membership check at `gates.py:271` and `gates.py:278`. Plain Python `set` membership check on raw strings. When the plan declares absolute paths and the agent declares project-prefixed relative paths, the strings never match.
- **Component B** — `_resolve_deposit_path` Strategy 0 cannot remap absolute paths to worktree. `os.path.join(wt_path, absolute_path)` returns `absolute_path` unchanged (Python `os.path.join` semantics), so the worktree-first check at `gates.py:228-235` is silently bypassed when the input is absolute.

Order of operations is correct: `gates.check()` at `bellows.py:485` runs BEFORE `_teardown_worktree` at `bellows.py:514`. No race exists. Fix shape is pure normalization, no order-of-operations change.

## Recursion-risk constraint (LESSONS 2026-05-19, reinforced 2026-05-27)

This plan modifies `_gate_deposit_exists`. The current (pre-fix) code performs the raw string comparison that is the bug being fixed. The fix plan's `**Deposits:**` blocks therefore use **project-prefixed relative paths** (`bellows/...`) to match the form agents declare in their Output Receipts. Authoring with absolute paths would trip the very bug being fixed during the QA step's gate evaluation.
