## Context

Phase 3 reproduction tested whether the agent terminates early on a 2-task plan (write marker, move to Done). It did not. Both opus and sonnet completed both tasks. This refuted the "agent terminates after first substantive action" hypothesis for simple plans.

But this morning's sanity check stranded with 4 substeps:
1. Write a new file (`bellows-sanity-check-2026-04-15.md`)
2. **Append** to an existing file (`agent-prompt-feedback.md`)
3. **Run a shell command** (`git add ... && git commit ...`)
4. `shutil.move` to Done

After the strand, only substep 1 had executed. The marker file existed; nothing else did.

Phase 3's reproduction did not include substeps 2 or 3. The agent's behavior on append-to-existing-file and shell-subprocess actions was NOT tested. This phase tests it.

**Why this matters before Phase 4:** Phase 3's recommended fixes (parser fix + planner consultation resilience + escalation strand check) address the multi-step strand mechanism. They do NOT address single-step strands. If single-step strands have a separate cause, Phase 4 needs a separate fix for them — or we ship Phase 4 and end up back here in a week investigating again.

Two outcomes possible:

- **REPRO_SUCCESS** (we reproduce the strand on a 4-substep plan) → there's a second bug. Phase 4 needs to address it. Most likely candidate: agent terminates after the shell-subprocess action (git commit) treats it as the "completion" signal, or some intermediate step throws an error the agent absorbs silently.
- **REPRO_FAIL** (the agent completes all 4 substeps cleanly) → this morning's strand was likely environmental noise (transient API issue, single occurrence). Ship Phase 4 as Phase 3 recommends and observe.

This phase is read-only and reproduction-only. No source code edits, no Bellows changes.
