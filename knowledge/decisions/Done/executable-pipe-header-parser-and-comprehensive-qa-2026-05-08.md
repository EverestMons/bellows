- Extend `_parse_plan_header()` to ALSO recognize the pipe-separated bold-Markdown header format on the second non-empty line of the file. The standard shape: `**Key:** value | **Key:** value | ...`. - Backward compatibility is required. YAML frontmatter must continue to work for the few plans (4 of 631 historical) that already use it. The parser tries YAML first; if YAML produces an empty dict OR no frontmatter is found, fall back to pipe-separated extraction.
- The agent's note about Fix A inertness was correct — the parser change is the gate. After this ships, every new pipe-separated plan with `pause_for_verdict: after_step_1` in its header will pause Bellows correctly between steps.
