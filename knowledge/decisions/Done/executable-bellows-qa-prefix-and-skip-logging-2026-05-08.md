Two related changes addressing the silent plan-pickup failure diagnosed earlier today (findings: `bellows/knowledge/development/diagnostic-plan-pickup-failure-findings-2026-05-08.md`):

1. **Add `qa-` to the dispatch whitelist regex.** The Planner deposits QA-only re-verification plans with the `qa-` prefix. Currently rejected silently by `is_runnable_plan()` at `bellows.py:709`. The diagnostic confirmed this is the root cause of today's pickup failure.
2. **Add a log line for silent skips.** When a `.md` file in a watched `decisions/` directory fails the prefix filter and is NOT a known lifecycle prefix (`in-progress-`, `verdict-pending-`, `halted-`), emit a one-time visible log line so future filter mismatches are debuggable instead of silent.

The new logging in change #2 will produce noise on these files; the implementer must include `roadmap-` in the lifecycle-prefix exemption list to suppress that noise.
- Secondary observations S2 (Step 2 auto-advance) and S3 (verdict-resolved retry loop) — separate diagnostic warranted.
