## Context

The 2026-05-22 `diagnostic-qa-step-detection-audit` (Done/) found that `_gate_is_qa_step` is a naked substring check (`"qa" in match.group(0).lower()` at `gates.py:543-549`) against the `## STEP N` header line. Population audit across 10 watched projects identified six leak patterns where the keyword fallback fails — "Invoice Security & Testing Analyst", "INVOICE SECURITY TESTING ANALYST", "INVOICE_SECURITY_TESTING_ANALYST", bare `## STEP N` with role in body, "Bellows Security & Testing", and descriptive headers like "Canary verification" with `**Agent:** QA` in the body. The fix-shape at `knowledge/research/qa-step-detection-fix-shape-2026-05-22.md` proposes a `qa_steps` plan-header field (comma-separated integers, e.g., `qa_steps: 2,4`) as the authoritative source, with the current keyword detection preserved as a backward-compatibility fallback.

**Why this matters under Rule 25:** PLANNER_TEMPLATE v4.48 routes the Planner to substance check (b) ONLY when both `rule_22_verification` and `rule_20_self_check` show PASS. A false `is_qa_step = False` produces vacuous PASS results for both QA-specific gates, which Rule 25 treats as clean — silently suppressing Planner-side verification of QA-step deliverables. The leak is invisible at gate time and only catchable via Planner-side substance review, which Rule 25 explicitly skips.

**Scope:** This plan ships only the code change in `gates.py` plus tests. The governance edit to PLANNER_TEMPLATE.md (header example, definitional paragraph, Gate 6 table description) ships as a separate plan in a follow-up session. The keyword fallback means the code change is safe to ship first — all in-flight and historical plans continue to behave exactly as today until the governance edit lands and Planner-authored plans start declaring `qa_steps`.

**Pre-existing test failures expected (5 carry-overs, not regressions):** 4 × `test_decisions.py::TestLoadPhrases` and `TestExtractDecisionBlocks` (worktree-environment artifact — INTERMEDIATE_DECISION_PHRASES.md missing at worktree root), 1 × `test_runner_parser.py::test_run_step_timeout` (BACKLOG-documented runner mock mismatch).

**Sources:** SA fix-shape at `bellows/knowledge/research/qa-step-detection-fix-shape-2026-05-22.md`; mechanism characterization at `bellows/knowledge/research/qa-step-detection-mechanism-2026-05-22.md`; population audit at `bellows/knowledge/research/qa-step-detection-population-audit-2026-05-22.md`.
