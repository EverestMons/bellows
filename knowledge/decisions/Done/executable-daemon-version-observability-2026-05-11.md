## Background

Closes the 2026-05-11 BACKLOG entry on daemon code-version observability. Bellows does not hot-reload Python modules; code changes require a manual restart, and there is no proactive observability that surfaces the loaded code version. Today's `deposit_exists` audit found three 2026-05-07/08 reproductions where the daemon ran pre-fix code despite the fix having shipped to disk — detecting this required inferring from gate-failure patterns + commit timestamps. This plan adds a startup log line and periodic heartbeat surfacing of module identifiers (git short SHA when available, mtime fallback) for the five core modules: `bellows.py`, `gates.py`, `verdict.py`, `parser.py`, `runner.py`.
